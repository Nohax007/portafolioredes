Práctica 4: ESP32 y Thingspeak

Noé Aguirre Gallegos, Diego Horcasitas Vera y Fernando Diaz Abascal
Ingeniería Mecatrónica
Fecha: Febrero 24, 2025
ESP32 Thingspeak

- Resumen -

Se implementa un sistema IoT con ESP32 que integra un sensor PIR, un sensor DHT22, un botón y un potenciómetro. Los datos obtenidos se transmiten a ThingSpeak para su visualización y análisis en tiempo real.

- Introducción -

El Internet de las Cosas (IoT) está revolucionando la manera en la que interactuamos con el entorno, permitiendo la conexión e integración de dispositivos físicos a redes digitales para el monitoreo, análisis y control en tiempo real. En esta práctica, se utiliza el microcontrolador ESP32, reconocido por su versatilidad y capacidad de procesamiento, para centralizar la adquisición de datos de diversos sensores y dispositivos de entrada. Entre ellos se incluyen un sensor PIR para la detección de movimiento, un sensor DHT22 para medir temperatura y humedad, un botón y un potenciómetro, los cuales se tratan en conjunto como fuentes de información ambiental y de interacción del usuario.

El objetivo principal es recopilar datos en tiempo real y transmitirlos a la plataforma ThingSpeak, facilitando la visualización y el análisis remoto mediante gráficos y otras herramientas de interpretación. ThingSpeak se posiciona como una solución ideal para proyectos IoT gracias a su facilidad de uso, integración con MATLAB y capacidad para manejar múltiples canales de datos simultáneamente. Además, el uso de un ESP32 en conjunto con estos sensores permite experimentar con la programación y la interconexión de hardware y software, aspectos fundamentales en el desarrollo de aplicaciones IoT modernas.

Esta práctica no solo refuerza conceptos básicos de electrónica y programación en entornos embebidos, sino que también brinda una experiencia práctica en la implementación de sistemas de monitoreo ambiental. La integración de sensores variados permite evaluar tanto la precisión de la adquisición de datos como la eficiencia en la transmisión y visualización de la información en la nube. Asimismo, se abordan temas de comunicación inalámbrica, manejo de datos y desarrollo de interfaces gráficas, elementos cruciales para la construcción de soluciones tecnológicas avanzadas.

La elección del ESP32 se debe a su conectividad WiFi integrada, lo cual simplifica la conexión a internet y la comunicación con plataformas como ThingSpeak. De igual manera, el uso del sensor DHT22 y el PIR provee una combinación robusta para evaluar tanto condiciones ambientales como la presencia o movimiento en un área determinada. El botón y el potenciómetro, por otro lado, aportan una dimensión interactiva que permite simular condiciones de entrada manual, abriendo la puerta a aplicaciones más complejas en el ámbito del control y la automatización.

Esta práctica se apoya en diversas fuentes de información técnica y tutoriales especializados, lo que facilita el entendimiento y la implementación de cada componente en el sistema IoT. La documentación oficial del ESP32, los tutoriales de Adafruit para el DHT22 y los recursos de ThingSpeak constituyen pilares esenciales para el éxito del proyecto, proporcionando el conocimiento necesario para integrar y optimizar cada parte del sistema.

- Materiales -

Los materiales utilizados para esta práctica son los siguientes:

  • ESP32
  • Thinspeak
  • Potenciometro
  • LED
  • Push button
  • Sensor de temperatura y humedad (DHT22)
  • Sensor de movimiento (PIR)
  • Resistencias (de 220 para el led y 10k ohms para el boton).

- Desarrollo -

Se desarrollaron primero las simulaciones en WOKWI donde se hicieron las conexiones a manera digital junto con su programación para comprobar el correcto funcionamiento del circuito y sus integrados.

Después se implementó a fisíco con la materiales antes dichos, usando ESP32, potenciometro, 2 resistencias, push button, led, PIR y el DHT22.

- Conclusiones -

  • La integración de múltiples sensores (PIR, DHT22, botón y potenciómetro) con el ESP32 permitió demostrar la versatilidad y eficacia de los sistemas IoT para la adquisición de datos en tiempo real.
  • La práctica evidenció la importancia de una correcta configuración y sincronización entre hardware y software para garantizar la recolección y transmisión de información de manera estable y confiable.
  • El uso de la plataforma ThingSpeak facilitó la visualización y el análisis de los datos, resaltando su utilidad para el monitoreo remoto y el seguimiento de variables ambientales e interactivas.
  • Se adquirieron conocimientos fundamentales sobre comunicación inalámbrica, protocolos de transmisión de datos y manejo de múltiples dispositivos, lo que resulta aplicable en el desarrollo de proyectos IoT más complejos.
  • La experiencia permitió identificar y resolver desafíos técnicos relacionados con la integración de componentes, contribuyendo al aprendizaje práctico y a la mejora continua en el diseño de soluciones embebidas.
