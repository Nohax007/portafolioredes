Práctica 6: Configuración de servidor para Red Local

IoT XAMPP

- Resumen -

- Introducción -

La configuración de un servidor local permite simular un entorno de alojamiento web sin necesidad de contar con un servidor externo. Esta práctica es común en entornos de desarrollo, educación o redes pequeñas. XAMPP es una de las herramientas más utilizadas para este fin.

Instalación de XAMPP y habilitación de servicios

XAMPP es un paquete que incluye Apache, MySQL, PHP y Perl. Para configurar un servidor local:

  1. Se descarga desde su sitio oficial.
  2. Se instala en el equipo.
  3. Se habilita el servicio Apache desde el panel de control de XAMPP.
  4. Se coloca el sitio web en la carpeta "htdocs" para ser servido localmente.

Configuración del firewall y acceso al puerto 80

El firewall de Windows u otros sistemas puede bloquear el acceso al servidor. Se debe permitir el acceso al puerto 80, que es utilizado por Apache para servir páginas web.

Identificación de la IP del servidor

Para que otros dispositivos accedan al sitio web local, deben estar conectados a la misma red. La IP local del equipo servidor se obtiene mediante el comando "ipconfig" en Windows. Esta IP se usa como URL en otros dispositivos.

Comprobación de acceso desde otros dispositivos

Con la IP del servidor y los servicios en funcionamiento, otros dispositivos de la misma red pueden acceder al sitio ingresando la IP en el navegador. Esto permite validar la correcta configuración del servidor local.

- Materiales -

Los materiales utilizados para esta práctica son los siguientes:

  • 3 computadoras o laptops diferentes
  • 1 computadora o laptop administradora
  • XAMPP
