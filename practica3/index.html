Práctica 3: Wifi ESP32 - Thingspeak

Noé Aguirre Gallegos, Diego Horcasitas Vera y Fernando Diaz Abascal
Ingeniería Mecatrónica
Fecha: Febrero 17, 2025
ESP32 IoT

- Resumen -

En esta tercera práctica se realizó la conexión del ESP32 con Thingspeak mediante WiFin de manera que se capturaron datos en la aplicación gracias a la comunicación WiFi.

Se utilizaron los sensores de la anterior práctica para ello, el boton, el potenciometro, el DHT22 y el sensor de movimiento PIR.

- Introducción -

En el mundo de la automatización y el Internet de las Cosas (IoT), la recopilación, transmisión y análisis de datos en tiempo real se han convertido en aspectos fundamentales para el desarrollo de sistemas inteligentes. Dispositivos como el ESP32, gracias a su capacidad de conectividad Wi-Fi y Bluetooth, ofrecen una solución eficiente para la comunicación de sensores con plataformas en la nube. Entre estas plataformas, ThingSpeak se destaca por su capacidad para almacenar, visualizar y analizar datos en tiempo real mediante el uso de gráficos y procesamiento de información basado en MATLAB.

La integración del ESP32 con ThingSpeak ha sido ampliamente explorada en diversas aplicaciones, como la monitorización remota de entornos domésticos (Cloud-Based Smart Home Monitoring using ThingSpeak and ESP32 - ResearchGate, 2023) y el desarrollo de sistemas inteligentes para la detección de anomalías (Anomaly Detection and Classification Using ESP32 and ThingSpeak - Journal of the Institute of Industrial and Intelligent Engineers, 2024). Estas implementaciones demuestran cómo el uso de dispositivos IoT puede mejorar la eficiencia y confiabilidad en múltiples sectores, incluyendo la automatización industrial y la gestión energética.

En esta práctica, se busca comprender el funcionamiento de la plataforma ThingSpeak en conjunto con el módulo ESP32, permitiendo la recopilación y transmisión de datos de sensores en tiempo real. Para ello, se explorará la configuración del entorno de desarrollo en Arduino IDE (ESP32 HTTP POST Request using Arduino IDE - Microcontrollers Lab, 2023), la conexión a redes Wi-Fi, la lectura de sensores y el envío de información a la nube. A lo largo del desarrollo de la práctica, se trabajará con ejemplos que incluyen sensores digitales, analógicos e inteligentes, aplicando los conceptos fundamentales de IoT para establecer una comunicación eficiente entre hardware y software.

Esta actividad nos permitirá adquirir experiencia práctica en la implementación de sistemas IoT y el uso de plataformas en la nube para la monitorización y análisis de datos, brindando una base sólida para el desarrollo de futuros proyectos tecnológicos y de automatización.

- Materiales -

Los materiales utilizados para esta práctica son los siguientes:

  • ESP32
  • Led
  • Potenciometro
  • Push button
  • Sensor de temperatura y humedad (DHT22)
  • Sensor de movimiento (PIR)
  • Resistencias (de 220 para el led y 10k ohms para el boton).

- Desarrollo -

Se desarrollaron primero las simulaciones en WOKWI donde se hicieron las conexiones a manera digital junto con su programación para el encendido del led junto con los sesnores correspondientes para cada caso (boton, potenciometro, DHT22 y PIR).

Después se implementó a fisíco con la materiales antes dichos, usando ESP32, un led, push button, resistencias, los sensores correspondientes y el programa de arduino.

- Conclusiones -

Al completar esta práctica, el estudiante habrá adquirido competencias clave en la integración de hardware y software para proyectos IoT. La experiencia práctica en la configuración del ESP32 y su comunicación con ThingSpeak proporciona una base sólida para el desarrollo de sistemas de monitorización remota y análisis de datos en tiempo real. Además, la implementación de ejemplos con diferentes tipos de sensores refuerza la comprensión de la versatilidad y aplicabilidad del ESP32 en diversos entornos y aplicaciones.
