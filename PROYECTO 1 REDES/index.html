Proyecto de Redes: Sistemas de monitoreo de corrientes

Noé Aguirre Gallegos, Diego Horcasitas Vera y Fernando Diaz Abascal
Ingeniería Mecatrónica
Fecha: Marzo 31, 2025
ESP32 Thingspeak

- Resumen -

En el presente proyecto se utilizo el sensor SCT013 conectado a Arduino Uno debido a complicaciones directas con las ESP32, sin embargo se utilizo el mismo principio.

Se utilizaron componentes como el LM358, un OP AMP con el fin de filtrar y amplificar la señal medida del sensor de corriente, los datos se extrajeron por la placa Arduino Uno y fueron pasados al ESP32 por medio de comunicación I2C para que de ahí se subieran a Thingspeak, donde se pueden visualizar los sistemas y como va la corriente.

- Introducción -

A lo largo de este proyecto se aborda el diseño e implementación de un sistema de medición y monitoreo de corrientes eléctricas empleando el sensor SCT013, el cual se encarga de convertir la corriente que circula por un conductor en una señal proporcional de fácil lectura. Aunque inicialmente se contempló la tarjeta ESP32 como la plataforma principal, surgieron complicaciones técnicas que motivaron el uso de Arduino Uno para la adquisición de datos. No obstante, el principio de medición se mantuvo y se estableció una comunicación I2C entre el Arduino Uno y la ESP32 para transferir los valores obtenidos.

Para lograr una señal estable y representativa, se incluyó un amplificador operacional LM358 que filtra y amplifica la salida del SCT013. De esta manera, el Arduino Uno puede leer con mayor precisión las variaciones de corriente. Posteriormente, los datos se envían a la ESP32 mediante I2C, y finalmente se publican en la plataforma Thingspeak, donde pueden visualizarse en tiempo real.

Este enfoque no solo demuestra la flexibilidad y robustez de integrar diferentes plataformas de hardware, sino que también pone de relieve la importancia de la adquisición y el análisis de datos de manera remota. A través de este sistema, es posible supervisar el consumo de corriente de equipos o instalaciones de forma continua, facilitando la toma de decisiones informadas y mejorando la eficiencia energética en aplicaciones residenciales, comerciales o industriales. Con ello, el proyecto se erige como un ejemplo de la convergencia entre la electrónica, la programación y la conectividad en la era del Internet de las Cosas (IoT).

- Materiales -

Los materiales utilizados para este proyecto son los siguientes:

  • ESP32
  • Sensor de corriente (SCT013)
  • Arduino Uno
  • LM358
  • Jumpers
  • Thingspeak

- Desarrollo -

Se desarrollaron primero distintas investigaciones con el fin de comprender completamente los sistemas a utilizar, despues de ello se paso a la etapa del diseño del sistema, donde se estuvo armando y diseñando al mismo tiempo, comprobando funcionamientos, al final se concluyo el sistema que terminamos de armar, incluyendo la Arduino Uno para extraer los datos y pasarlos al ESP32 para subrilos a Thingspeak.

Se uso una programación Maestro-Esclavo con las placas Arduino Uno y la ESP32, a manera que se utilzó como Maestro la placa Arduino Uno que es la que extrae y manda los datos al ESP32.

El ESP32 fue el esclavo, recibiendo los datos mediante comunicación I2C con la Arduino y el ESP32 solo lee y sube los datos a Thinspeak, de forma que podamos monitorear las corrientes de distinas aulas del IDIT.

Al ser 5 personas en el equipo, existen actualmente 5 dispositivos midiendo las corrientes de 5 distintos salones.

- Conclusiones -

  • Validación del enfoque de medición: La práctica demuestra que es viable emplear el sensor SCT013 para la medición de corriente, utilizando un amplificador operacional LM358 para filtrar y amplificar la señal, lo que garantiza lecturas precisas y estables.
  • Integración de plataformas: Aunque inicialmente se presentaron dificultades con el uso directo de la ESP32, la implementación del Arduino Uno como dispositivo intermediario evidenció la capacidad de adaptación y la efectividad de la comunicación I2C para transferir datos de forma confiable.
  • Visualización y monitoreo en tiempo real: La transferencia de datos a través de la ESP32 a la plataforma ThingSpeak permite visualizar el comportamiento del consumo eléctrico en tiempo real, facilitando la supervisión y análisis de los sistemas monitorizados.
  • Flexibilidad y robustez del sistema: El proyecto resalta la importancia de contar con alternativas en el diseño de sistemas de adquisición de datos, demostrando que la integración de distintos dispositivos (Arduino Uno y ESP32) puede superar limitaciones técnicas y ofrecer soluciones robustas para aplicaciones de IoT.
  • Aplicaciones prácticas y futuras mejoras: Los resultados obtenidos abren la puerta a la aplicación del sistema en entornos residenciales, comerciales e industriales. Como recomendaciones para futuras iteraciones, se sugiere optimizar el filtrado de la señal, mejorar la eficiencia de la comunicación y explorar algoritmos de análisis avanzado para incrementar la funcionalidad y la precisión del sistema.
