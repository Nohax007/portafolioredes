Práctica 7: Redes de computadora con Switches y Cables

Noé Aguirre Gallegos, Diego Horcasitas Vera y Fernando Diaz Abascal
Ingeniería Mecatrónica
Fecha: Marzo 7, 2025
IoT Redes

- Resumen -

Se hicieron distintas redes con 3 computadoras, un switch y un router, de forma que se hizo una red alambrica con el switch, inalambrica con el router y se hizo una red conjunta de estas dos.

Para ello se utilizaron cables de ethernet y para comprobar los dispositivos se hizo ping a cada uno de los dispositivos con la dirección IP asignada de manera automatica y obtenida en la terminal con el comando ipconfig.

- Introducción -

Las redes de computadoras en entornos locales (LAN) permiten la interconexión de múltiples dispositivos para compartir recursos, servicios y datos. La práctica con switches y routers es fundamental para entender la estructura de redes pequeñas y medianas.

Funcionamiento del switch y del router

El switch es un dispositivo de red que conecta múltiples computadoras dentro de una red local. Su función principal es recibir, procesar y reenviar datos hacia el dispositivo de destino. Opera en la capa 2 del modelo OSI (enlace de datos).

El router, por otro lado, conecta diferentes redes entre sí y se encarga de dirigir el tráfico entre ellas. Opera en la capa 3 del modelo OSI (red) y asigna direcciones IP a los dispositivos conectados.

Diseño de red con switch

Para configurar una red con un switch:

  1. Se utilizan cables de red directos.
  2. Se conecta el switch a un mínimo de 3 computadoras.
  3. Una de las computadoras funciona como servidor local.
  4. Se usa la asignación automática de IP (DHCP).
  5. Se verifica la conectividad mediante el comando PING y el acceso al sitio web del servidor.

Diseño de red con router

Para configurar una red con un router:

  1. Se utiliza un router con capacidad para conexión alámbrica e inalámbrica.
  2. Se conecta al menos 3 computadoras, una de ellas como servidor.
  3. Se asignan IP automáticamente a través del router.
  4. Se comprueba la comunicación entre dispositivos y el acceso al servidor.

Unión de ambas redes

Para integrar las redes con switch y router:

  1. Se identifican los cables adecuados (puede requerirse cable cruzado para ciertas conexiones).
  2. Se habilita la asignación automática de IP.
  3. Se comprueba la conectividad total mediante PING y el intercambio de archivos entre computadoras de ambas redes.

- Materiales -

Los materiales utilizados para esta práctica son los siguientes:

  • 6 computadoras
  • 1 switch
  • 1 router
  • Cables de red directos y cruzados

Se conectan las computadoras al switch de manera que se hace la primer red con direcciones IPv4 que se proporcionan de manera automatica.

Se hace Ping en cada dispositivo usando la dirección IPv4 dada para comprobar la conectividad.

Para la segunda red usamos la conexión inalambrica, de manera que nos conectamos al router sin cables Ethernet.

Igual se compruba su funcionamiento al usar el comando ping en cada dispositivo.

Finalmente se juntan ambas redes en una sola para la última red.

Se comprueba la conectividad entre ambas redes usando nuevamente el comando ping en cada dispositivo.

- Conclusiones -

  • Comprensión integral de los dispositivos de interconexión: La práctica demostró que, aunque el switch y el router funcionan en capas diferentes del modelo OSI (capa 2 y capa 3, respectivamente), ambos son esenciales y complementarios. El switch optimiza el tráfico dentro de la LAN segmentándolo por direcciones MAC, mientras que el router permite la comunicación entre redes y administra los rangos de direcciones IP. Al configurar y observar su operación se constató cómo cada dispositivo desempeña un rol específico e imprescindible para la conectividad eficiente.
  • Importancia de la planificación física y lógica: Seleccionar el tipo de cable correcto (directo o cruzado) y la topología adecuada resultó decisivo para evitar cuellos de botella y pérdidas de conectividad. El uso de cables directos para conectar hosts al switch y, cuando fue necesario, cables cruzados para la interconexión switch-router permitió establecer enlaces estables y sin errores, subrayando la necesidad de una planeación cuidadosa tanto del hardware como del direccionamiento IP.
  • DHCP como facilitador de escalabilidad y gestión: La configuración con asignación automática de IP evidenció las ventajas operativas del DHCP: reducción del trabajo manual, disminución de errores de asignación y rápida incorporación de nuevos dispositivos. Al comparar el tiempo requerido para agregar hosts en modo estático frente a DHCP, se corroboró la eficiencia y conveniencia de este servicio, sobre todo en redes pequeñas y medianas que pueden crecer con rapidez.
  • Verificación y diagnóstico con herramientas básicas: El uso sistemático de comandos como PING y la prueba de acceso a la página web alojada en el servidor local permitió validar, paso a paso, la correcta configuración de cada segmento de red. Estas pruebas sencillas se convirtieron en el primer filtro de diagnóstico para detectar fallos de cableado, errores de puerta de enlace o conflictos de IP, reforzando la importancia de dominar utilidades de red elementales antes de recurrir a herramientas más complejas.
  • Interoperabilidad y transferencia de datos entre subredes: La unión de las dos redes —la segmentada por switch y la administrada por router— demostró que, con parámetros coherentes (rango de IP, máscara, puerta de enlace), se logra una infraestructura unificada capaz de compartir servicios, archivos y aplicaciones sin percibir fronteras físicas. Esto resalta la relevancia de diseñar redes modulares que luego puedan integrarse sin grandes reconfiguraciones, favoreciendo la escalabilidad futura.
  • Refuerzo de las buenas prácticas de documentación: Registrar topologías, configuraciones, rangos de IP y pruebas de conectividad facilitó la detección de problemas y la explicación de la red a terceros. La práctica confirma que la documentación no es un trámite administrativo, sino un recurso técnico imprescindible para administrar, escalar o solucionar incidencias con agilidad.
